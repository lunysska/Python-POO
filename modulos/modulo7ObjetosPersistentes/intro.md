# 7. Objetos Persistentes

# 7.1. Persistencia
La persistencia de datos se refiere a la capacidad de una aplicación para guardar datos de forma que puedan recuperarse y
utilizarse posteriormente, incluso después de que la aplicación se haya cerrado o el sistema se haya reiniciado. 
Garantiza que los datos permanezcan intactos y accesibles en diferentes sesiones de la aplicación.

# 7.2 Què tipo de persistencia de datos podemos tener?
Por mencionar sòlo algunas:

- Guardar datos en un archivo
- Guardar tus datos en una base de datos de dispositivos local
- Envío de datos a un servidor
- Guardar datos en un dispositivo de almacenamiento externo, como una memoria USB.
- Guardar tus datos en una base de datos remota
- Imprimir sus datos

# 7.3. Manejo de archivos CSV en Python

## 7.3.1 Que es un archivo CSV?

Un archivo CSV (Comma Separated Values o valores separados por comas) es un formato de texto plano sencillo para almacenar datos estructurados, como tablas o hojas de cálculo. Cada línea representa una fila y cada valor está separado por una coma (o punto y coma), permitiendo intercambiar información fácilmente entre bases de datos y programas como Excel.

Características y detalles clave:

    - Estructura: Son texto simple; cada línea es un registro y cada campo se separa por un delimitador, usualmente comas, aunque también pueden ser puntos y coma, tabulaciones o espacios.
    - Uso: Ideal para mover grandes volúmenes de datos entre aplicaciones distintas, importar/exportar contactos, o manejar bases de datos simples.
    - Ventajas: Es un formato universal, ligero, legible por humanos y compatible con casi cualquier software de hojas de cálculo (Excel, Google Sheets).
    - Sinónimos: Se le conoce frecuentemente como archivo de texto, archivo de valores delimitados o, simplemente, fichero de datos.
